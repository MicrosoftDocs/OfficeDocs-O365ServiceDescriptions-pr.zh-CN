# <a name="microsoft-project-service-description"></a>Microsoft 项目服务说明

## <a name="overview"></a>概述

Microsoft Project 提供以下应用程序，以帮助满足组织对项目和工作管理的需求：

- Project 网页版
- Project Online 
- Project Online 桌面客户端

本文将帮助你了解每个 Project 订阅中提供哪些应用程序以及每个应用程序提供哪些功能。

### <a name="project-for-the-web"></a>Project 网页版

Project 网页应用是 Microsoft 最新的基于云的工作和项目管理产品。 Project 网页项目提供简单、强大的工作管理功能，以满足大多数需要和角色。 项目经理和工作组成员可以使用 Project 网页项目来计划和管理任何规模的工作。

Project 网页应用基于 Microsoft Power Platform 构建。 Power Platform 由 PowerApps、Power Automate、Power BI 和 Microsoft Dataverse 组成。 Project 网页数据存储在 Microsoft Dataverse 中。 

Project 网页版通过以下三个订阅提供：项目计划 1、项目计划 3 和项目计划 5。 Project [网页功能中列出的功能仅](#project-for-the-web-features) 与 Project 网页数据存储中的数据进行交互。

### <a name="project-online"></a>Project Online
 
Project Online 是一种灵活的联机解决方案，适用于 PPM (和) 项目组合管理。 Project Online 提供了强大的项目管理功能，用于几乎任何设备上的任何位置规划、确定项目和项目组合投资优先级 &mdash; 以及管理项目组合投资。 Project Online 适用于管理员、项目组合经理、项目组合查看者、项目经理、资源经理、工作组领导和成员。

Project Online 基于 SharePoint 平台构建，它将数据存储在 SharePoint 数据存储中。

Project Online 通过两个订阅提供：项目计划 3 和项目计划 5。 [Project Online 功能中列出的功能](#project-online-features)仅与 Project Online SharePoint 数据存储中的数据进行交互。

Project Online 的团队成员功能通过 Project Online Essentials 或 Project 计划 1 订阅提供。 Project Online Essentials 是具有 Project 计划 3 或项目计划 5 订阅的客户的团队成员附加订阅。 Project Online Essentials 和 Project Plan 1 订阅为工作组成员提供了一个 Web 界面，用于对 Project Online 中管理的项目执行任务和时间更新。

具有 Project Online Essentials 或 Project Plan 1 订阅的团队成员可以执行以下 Project Online 功能：

- 使用基于 Web 的界面
- 更新任务、问题和风险
- 提交时间表
- 共享文档并与 Microsoft Teams 或 Skype for Business 用户协作

Project Online Essentials 不包括 Project 网页版。 项目计划 1 包括 Project 网页版和 Project Online Essentials。

### <a name="project-online-desktop-client"></a>Project Online 桌面客户端

Microsoft Project Online 桌面客户端是一种项目管理计划，具有自动计划、项目资源管理和内置报告功能。 Project Online 桌面客户端可以用作独立应用程序，也可以连接到 Project Online。 Project Online 桌面客户端是 Project Professional 桌面客户端的订阅版本，可用于 Project 计划 3 和 Project 计划 5 订阅。

### <a name="using-project-for-the-web-and-project-online-together"></a>将 Project 网页版和 Project Online 一同使用

虽然 Project 网页版和 Project Online 是单独的应用程序，但可以并排使用它们。 可以在 Project Home 中查看这两个应用程序中的所有项目。 可以使用 Project Roadmap 功能生成包含两个应用程序中的项目的路线图。 此外，您还可以使用 Power BI 和 Project 网页 Power BI 内容包，深入了解这两个应用程序中的项目和资源。<sup>3</sup>

此图显示了应用程序如何并行协作。

![显示 Project 网页版和 Project Online 的图表](../media/project-online-applications.png)

今后，Microsoft 将专注于 Project 网页创新。 但是，您可以放心地继续使用 Project Online，因为它会继续获得关键性能和安全改进。

## <a name="service-considerations"></a>服务注意事项

> [!NOTE]
> Project 网页应用当前对 GCC、GCC High 和 DoD 不可用。 我们正致力于将 Project 网页发布给你，但目前无法共享一个稳固的时间范围。

### <a name="licensing-considerations"></a>许可注意事项

- Project Online 网站上的任何交互都至少需要租户内的项目计划 3 或项目计划 5 订阅。
- 当最后一个项目计划 1、项目计划 3 或项目计划 5 订阅过期时，Web 实例的 Project 将不会自动删除，除非没有依赖于 Microsoft Dataverse 的活动订阅。
- 当最后一个项目计划 3 或项目计划 5 订阅过期时，Project Online 实例将在 120 天后删除。
- 对于 Project Online 试用订阅，试用版将在试用期结束后的 30 天内删除。
- 对于 Project 网页版试用订阅，试用版实例不会被删除，除非没有依赖于 Microsoft Dataverse 的活动订阅。

### <a name="use-of-sharepoint-online"></a>使用 SharePoint Online

Project Online 需要使用 SharePoint Online，它作为 Project Online 的一部分进行预配。 对项目计划 3 或项目计划 5 订阅提供的 SharePoint Online 功能的访问权限仅限于存储和访问数据以支持 Project Online。

### <a name="project-roadmap-and-power-automate"></a>Project 路线图和 Power Automate

Project 路线图需要使用 Power Automate，它作为 Project 订阅的一部分进行预配。 Power Automate 功能权限仅限于 Project 路线图所需的 Power Automate 功能。 Project 路线图所需的 Power Automate 功能在 Microsoft 365 管理中心的"应用"部分显示为 **"Project** 与 Flow 的数据集成"服务计划。

### <a name="project-for-the-web-and-microsoft-dataverse"></a>Project 网页和 Microsoft Dataverse

Project 网页应用需要使用 Microsoft Dataverse 来存储其数据。 Microsoft Dataverse 数据库作为 Project 订阅的一部分进行设置。 对 Microsoft Dataverse 功能的访问权限仅限于存储和访问数据以支持 Project 网页版。 Project 所需的 Microsoft Dataverse 功能在 Microsoft 365 管理中心的"应用"部分显示为 Project 服务计划的 Common **Data** Service。

对于具有 5 (5) 或多个 Project 网页版许可证的 Project 客户，适用于 Web 客户的 Project 可以部署到 Power Platform Production 和 Sandbox 环境。
