# <a name="insights"></a>见解

SharePoint Online 是基于 web 的工具和技术的集合，可帮助您的组织存储、共享和管理数字信息。 此托管服务基于 Microsoft SharePoint Server 2013 构建，是处理项目、在中心位置存储数据和文档以及与他人共享信息的理想服务。 以下"洞察力"功能可让组织使用数据库、报表和业务应用程序中的信息来解决具体的业务需求。
  
## <a name="business-intelligence-center"></a>商业智能中心

商业智能中心网站允许您将数据连接、工作簿、报告、记分卡和仪表板存储在一个安全的中心位置。 详细了解如何[配置商业智能中心](https://go.microsoft.com/fwlink/p/?LinkId=270883)。
  
## <a name="calculated-measures-and-members"></a>计算度量值和成员

如果创建数据透视图报表的数据透视表报表，且报表使用的数据存储在 SQL Server Analysis Services 中，可以为报表创建计算度量值和计算成员。详细了解 [Excel Services 中的商业智能功能](https://go.microsoft.com/fwlink/p/?LinkId=271025)。
  
## <a name="data-connection-library"></a>数据连接库

数据连接库是一种特殊的 SharePoint 库，旨在存储和管理共享数据连接。 这些包括使用 Excel 创建的 ODC 文件。 了解有关[SharePoint Server 2013 中的受信任数据连接库的](https://go.microsoft.com/fwlink/p/?LinkId=271026)详细信息。
  
## <a name="decoupled-pivottables-and-pivotcharts"></a>分离的 PivotTable 和 PivotChart

创建数据透视图报表后，会自动创建相应的数据透视表报表。 在 Excel 2013 中，无需在与对应的数据透视图报表相同的工作表上显示数据透视表。 可以选择仅显示数据透视图报表。 此功能称为*分离的数据透视表和数据透视图*。 详细了解[数据透视表和数据透视图](https://go.microsoft.com/fwlink/?LinkId=271027)。
  
## <a name="excel-services"></a>Excel Services

Excel Services 允许您与其他人共享工作簿、与报表交互、在浏览器窗口中查看和浏览信息以及刷新数据。 Excel Services 还允许您与其他人共享全部或部分工作簿，并控制这些工作簿中显示的信息。 在 SharePoint 2016 中，Excel Services 功能可通过 Office Online Server 上的 web Excel 获取。 Excel Services 需要 Office Online Server 必须位于本地环境中。 详细了解 [Excel Services](https://go.microsoft.com/fwlink/p/?LinkId=271028)。
  
## <a name="excel-web-access-web-part-for-sharepoint"></a>SharePoint 相关的 Excel Web Access Web 部件

您可以使用 Excel Web Access Web 部件显示 SharePoint 网站上的 Excel 内容。
  
## <a name="field-list-and-field-support"></a>字段列表和字段支持

Excel for the web 允许您打开字段列表和字段，以便在浏览器窗口中查看数据透视图报表和数据透视表报表。 借助此功能，无需打开 Excel，即可方便地暂时更改数据透视图报表或数据透视表报表中显示的信息。 了解有关[Excel Services 中的商业智能的](https://go.microsoft.com/fwlink/p/?LinkId=271029)详细信息。
  
## <a name="filter-enhancements"></a>筛选器增强功能

可更方便地创建切片器并将其连接至报告。您可以调整切片器的大小和方向。也可通过应用内置样式或定义自己的样式来格式化切片器。
  
## <a name="filter-search"></a>筛选器搜索

可以搜索筛选结果，这样就可以轻松跳到长列表中的特定项。详细了解如何[使用筛选器和切片器](https://go.microsoft.com/fwlink/p/?LinkId=271030)。
  
## <a name="odc-file-support"></a>ODC 文件支持

ODC 文件使您可以管理、共享和重复使用与企业数据源的数据连接。 特定数据源的数据连接可以由 IT 人员集中配置和管理，并使其在任何使用 ODC 文件的工作簿中可用。 您可以在 SharePoint 服务器场上的中央位置维护 ODC 文件集，并且可以编辑 Excel 工作簿以使用这些文件。
  
## <a name="performancepoint-services"></a>PerformancePoint Services

对 SharePoint Online 客户不可用。 PerformancePoint Services 允许 SharePoint Server 2013 客户创建交互式仪表板，以显示记分卡、报表和筛选器形式的关键绩效指标（Kpi）和数据 virtualizations。 详细了解 [PerformancePoint Services](https://go.microsoft.com/fwlink/p/?LinkId=271032)。
  
## <a name="power-bi-web-part"></a>Power BI web 部件

Power BI 是一个基于云的业务分析工具套件，使任何人都能够以更快、更高的效率和理解方式连接、可视化和分析数据。 通过简单易用的仪表板将用户连接到各种实时数据，提供交互式报表，并提供极具吸引力的可视化效果，使数据更加生动。 若要了解 Power BI 的详细信息，请参阅 [Get started with Power BI service](https://go.microsoft.com/fwlink/?linkid=844671)（Power BI 服务入门）。
  
## <a name="power-pivot-for-excel-in-sharepoint"></a>SharePoint 中适用于 Excel 的 Power Pivot

借助 Power Pivot for Excel，可以执行功能强大的数据分析，并能创建复杂的数据模型。 还可以混合来自各种来源的大量数据，迅速分析数据并共享见解。 使用 SharePoint Online，可以使用 Excel 来查看浏览器窗口中的图表、表、数据透视图报表和数据透视表，并与之交互。 还可以使用 Excel Web Access Web 部件在 SharePoint 网站中显示 Excel 内容。 必须先在 Excel 中创建 Power Pivot 表，然后才能在 SharePoint 中查看。 详细了解如何使用 [Excel 中的 Power Pivot](https://go.microsoft.com/fwlink/p/?LinkId=271036)。
  
## <a name="power-view-for-excel-in-sharepoint"></a>SharePoint 中适用于 Excel 的 Power View

 借助 Power View for Excel，可以使用高度交互式可视化效果、动画和智能查询来直观呈现模型化数据，并与之进行交互。 使用 SharePoint Online，可以使用 Excel 进行 web 查看，并与浏览器窗口中包含 Power View 表的工作簿进行交互。 还可以使用 Excel Web Access Web 部件在 SharePoint 网站中显示 Excel 内容。 必须先在 Excel 中创建 Power View 表，然后才能在 SharePoint 中查看。 详细了解如何使用 [Excel 中的 Power View](https://go.microsoft.com/fwlink/p/?LinkId=271035)。
  
## <a name="quick-explore"></a>快速浏览

通过快速浏览，组织可以选择数据透视图报表或数据透视表中的值，并查看有关该值的其他信息。 例如，如果要查看销售报表并选择表示总销售额的值，可以使用"快速浏览"来了解其他信息，如跨不同产品类别或地理区域的销售额。 详细了解[快速浏览](https://go.microsoft.com/fwlink/p/?LinkId=271037)。
  
## <a name="refresh-excel-workbooks-interactively"></a>以交互方式刷新 Excel 工作簿

允许您查看 Excel 中连接到外部数据源的 web 工作簿（例如，MS SQL 数据库）的最新数据。 请注意，您必须先将工作簿连接到 Excel 中的外部数据源，然后才能在 Excel for web 中查看它们。
  
## <a name="sql-server-reporting-services-ssrs-integrated-mode"></a>SQL Server Reporting Services （SSRS）集成模式

如果 SQL Server 2012 Reporting Services 与 SharePoint 集成，可以将强大的 SharePoint 协作和文档集中管理功能用于 SQL Server 2012 Reporting Services 内容。 报告存储在 SharePoint 文档库中，并与其他报告（如 Excel 中的 web 文件）一起存储。 SharePoint 权限可用于控制对 Reporting Services 内容的访问，用户可以在文档库中启动报表设计器、模型设计器和报表生成器。 了解有关[Reporting Services 报告服务器](https://go.microsoft.com/fwlink/p/?LinkId=271039)的详细信息。
  
## <a name="sql-server-reporting-services-ssrs-web-part"></a>SQL Server Reporting Services (SSRS) Web 部件

SQL Server 2012 Reporting Services Web 部件允许用户在 SharePoint 网站中的自己的 Web 部件中显示现有的 Reporting Services 报表。 如果 Reporting Services 报表包含参数，则用户可以使用 web 部件设置将筛选器连接到该报表。 详细了解[报表查看器 Web 部件](https://go.microsoft.com/fwlink/p/?LinkId=271041)。
  
## <a name="timeline-slicer"></a>日程表切片器

日程表切片器是可添加到使用 Excel 创建的仪表板中的控件。 使用日程表控件，用户可以查看特定时间段内的信息。 详细了解 [Excel Services 中的商业智能](https://go.microsoft.com/fwlink/p/?LinkId=271042)。
  
## <a name="use-and-interact-with-excel-workbooks-containing-data-models"></a>使用含有数据模型的 Excel 工作簿并与之进行交互

Excel 工作簿有时包含 Power Pivot 数据模型，你可以进行功能强大的数据分析、混搭大量不同来源的数据，迅速分析数据并共享见解。
  
## <a name="user-defined-functions"></a>用户定义的函数

用户定义函数（Udf）使您能够在 Excel 中将公式用于 web 工作表单元格，以调用在托管代码中编写的自定义函数。 Udf 是一项高级可编程性功能，允许企业为 web 工作表开发和插入 Excel 中的自定义代码。 您可以创建 Udf 以调用自定义数学函数、将数据从自定义数据源获取到工作表，以及从 Udf 调用 web 服务。
  
## <a name="visio-services"></a>Visio Services

Visio Services 是一种服务应用程序，允许用户共享和查看 Microsoft Visio\*绘图（.vsdx）和 Visio 2010 web 绘图\*（vdw）文件。 Visio Services 不再在 SharePoint Online 中可用。 Visio Services 将继续在 SharePoint Server 2010、SharePoint Server 2013 和 SharePoint Server 2016 中提供。 详细了解 [Visio Services](https://go.microsoft.com/fwlink/p/?LinkID=271043)。
